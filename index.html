Stream processing in IoT environment

is designed to handle streaming data at the edge of the network with low latency. Streaming data generated from IoT devices is often unstructured, such as video and audio streams, or structured but meaningless without regarding its temporal characteristics. Cloud based services are not generally good at processing these kinds of data. Preprocessing streams of data before they're sent to the cloud makes large scale data processing in the cloud more efficient and reduces the usage of the network bandwidth. To deal with such nature of data, SensorBee has following traits:

Stateful

Extracting useful information from unstructured data often requires machine learning. SensorBee is designed to work well with online machine learning algorithms as user-defined states (UDSs). Supported machine learning toolkit includes Chainer and some of algorithms implemented in Jubatus. It also provides means of stateful temporal data handling as a part of its query language.

Expressive

Operations and queries are written in BQL, a dialect of CQL (Continuous Query Language). BQL is a very powerful language and easy to learn for people familiar with SQL. BQL is schemaless and uses JSON-like data structure as its internal data structure. It also provides many features useful to deal with JSON.

Lightweight

SensorBee has small footprint, less than 30MB at minimum, and can be run on a small computer such as Raspberry Pi. Although it isn't small enough to be embedded into a very small device at the moment, one of the goals of SensorBee is to run BQL statements on a very weak chip.

Getting Started

Install Go

is written in Go and requires it to be installed in advance. Please look at the official installation guide to install Go tools and set up its development environment (i.e. setting GOROOT and GOPATH environment variables).

Install SensorBee

$ go get gopkg.in/sensorbee/sensorbee.v0/...
sensorbee and build_sensorbee commands are installed under $GOPATH/bin.

Run SensorBee

$ sensorbee run

Build Custom sensorbee Command

Since SensorBee is written in Go, all plugins are statically linked. When you want to add plugins to it, you need to build a custom sensorbee command. 
$ ls
build.xml
$ cat build.xml
plugins:
  - github.com/sensorbee/twitter/plugin
$ build_sensorbee
sensorbee_main.go
$ ls
build.yaml
sensorbee
sensorbee_main.go
